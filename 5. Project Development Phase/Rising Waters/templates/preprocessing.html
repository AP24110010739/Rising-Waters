Data Preprocessing Guide

Machine Learning Data Preprocessing

Before model training, raw meteorological records must be cleaned, encoded, and scaled. Explore the theoretical foundations and see how they are applied in our pipeline.

Step 3.1

Missing Value Detection

Handling missing values is an essential data preprocessing step because incomplete data compromises machine learning model accuracy and reliability. Missing data occurs when entries are empty or unrecorded, which can cause training errors or incorrect predictions.

To detect missing values, the function isnull().sum() is used. It checks each cell, returning True for missing and False for present, and counts the total missing values per column. The function isnull().any() confirms if any column contains missing data at all.

# Detect & sum nulls per column print(df.isnull().sum()) # Confirm if any null exists if df.isnull().any().any(): # Impute using column median df.fillna(df.median(), inplace=True)
Step 3.2

Outlier IQR Capping

Outlier handling is an important preprocessing step because extreme values can negatively affect machine learning model performance. Outliers are identified using box plots and the Interquartile Range (IQR) method, where values outside the calculated upper and lower bounds are considered outliers.

Instead of removing these values, the capping technique is applied to preserve the dataset size. Values above the upper limit are replaced with the upper bound, while values below the lower limit are replaced with the lower bound. In this project, outliers were detected and treated using the IQR-based capping method.

Q1 = df[col].quantile(0.25) Q3 = df[col].quantile(0.75) IQR = Q3 - Q1 lower = Q1 - 1.5 * IQR upper = Q3 + 1.5 * IQR # Cap outliers outside bounds df[col] = df[col].clip(lower, upper)
Step 3.3

Categorical Value Encoding

Categorical features must be converted into numerical format before model training because machine learning algorithms work mainly with numbers. Raw text labels will cause errors during training if passed directly.

Feature Mapping: Used when categories have a clear order or fixed meaning (e.g. Low=0, Medium=1, High=2).
Label Encoding: Assigns a unique integer value to each unique string label (e.g. Male=0, Female=1). This uses Scikit-learn's LabelEncoder.

from sklearn.preprocessing import LabelEncoder le = LabelEncoder() # Fit & transform object columns df['State'] = le.fit_transform(df['State'])
Step 3.4 & 3.5

Dependent & Independent Variables

Splitting the dataset into dependent and independent variables is a critical step before model training. This step completely separates the input data from the output data, allowing the model to learn the relationship between them.

Independent Variables (X): Input features (meteorological readings like Cloud Cover, ANNUAL, etc.) used by the model to learn patterns and make predictions.
Target Variable (y): The dependent variable, containing prediction labels (like Floods class column).

# Independent features (X) X = df.drop('Floods', axis=1) # Dependent feature (y) y = df['Floods'] # Split into Train / Test (75-25) X_tr, X_te, y_tr, y_te = train_test_split( X, y, test_size=0.25, random_state=42 )
Step 3.6

Feature Scaling & Preventing Leakage

Feature scaling is an essential preprocessing step used to bring all input features into a similar range. Without scaling, algorithms may give more importance to features with larger numerical ranges.

We apply Scikit-learn's StandardScaler, which normalizes features with formula z = (x - μ) / σ so that mean is close to 0 and standard deviation is close to 1.

Critical Rule: Preventing Data Leakage! The scaler must be fitted ONLY on the training data (X_train) and then used to transform both training and testing datasets. Doing fit_transform() on test data leaks statistical distributions from the test set, creating optimistic bias and training bugs.

from sklearn.preprocessing import StandardScaler sc = StandardScaler() # Correct Implementation X_train_sc = sc.fit_transform(X_train) # fit + transform X_test_sc = sc.transform(X_test) # transform ONLY (no fit!) # Warning: sc.fit_transform(X_test) is a BUG!
