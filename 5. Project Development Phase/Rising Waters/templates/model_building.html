Epic 4 — Model Building

Machine Learning Model Building

Four classification algorithms are implemented as modular functions, trained, evaluated and compared. XGBoost is selected as the final deployment model.

Step 4.1 – 4.4   Algorithm Initialization

Library Imports & Initialization

Each classifier is imported from the scikit-learn library and xgboost. All four models are initialized and trained using the same preprocessed data splits for a fair comparison.

# Import algorithms from sklearn import tree from sklearn import ensemble from sklearn import neighbors import xgboost # Initialize classifiers dtree = tree.DecisionTreeClassifier() Rf = ensemble.RandomForestClassifier() knn = neighbors.KNeighborsClassifier() xgb = xgboost.XGBClassifier() # Train all models on the same preprocessed training data dtree.fit(x_train, y_train) Rf.fit(x_train, y_train) knn.fit(x_train, y_train) xgb.fit(x_train, y_train)
Modular Function Approach

Classification Algorithm Functions

A separate function is created for each model. This modular approach keeps the codebase organized, reusable, and easy to understand. Each function accepts X_train, X_test, y_train, y_test and returns the trained model and predictions.

Story 4.1

Decision Tree — decisiontree()

A Decision Tree is a supervised machine learning algorithm used for classification and prediction. It splits the dataset into branches based on feature values. Each branch represents a decision condition, and the final leaf node represents the predicted class.

  • 1Function decisiontree() initialized with X_train, X_test, y_train, y_test parameters.
  • 2DecisionTreeClassifier is initialized from scikit-learn.
  • 3.fit() trains the model — learns patterns between features and target.
  • 4.predict() generates predictions on unseen test data.
  • 5Confusion matrix, classification report, and accuracy score are printed.
def decisiontree(X_train, X_test, y_train, y_test): print("\n===== DECISION TREE MODEL BUILDING =====") # 1. Initialize model = tree.DecisionTreeClassifier() # 2. Train model.fit(X_train, y_train) # 3. Predict y_pred = model.predict(X_test) # 4. Evaluate accuracy = accuracy_score(y_test, y_pred) cm = confusion_matrix(y_test, y_pred) cr = classification_report(y_test, y_pred) print(f"\n[RESULT] Accuracy: {accuracy:.4f}") print("\nConfusion Matrix:\n"); print(cm) print("\nClassification Report:\n"); print(cr) return model, y_pred
Story 4.2

Random Forest — randomForest()

Random Forest is an ensemble method that trains many independent Decision Trees on random subsets of training data and features. Final prediction is made by majority vote. Reduces variance while maintaining low bias — giving a more stable, generalised model.

  • 1Function randomForest() defined with n_estimators=100, random_state=42 parameters.
  • 2RandomForestClassifier initialized with specified hyperparameters.
  • 3.fit() trains the ensemble of decision trees on X_train, y_train.
  • 4.predict() generates predictions using majority voting across all trees.
  • 5Accuracy score, confusion matrix, classification report printed. Returns model & predictions.
import numpy as np from sklearn.ensemble import RandomForestClassifier from sklearn.metrics import confusion_matrix, classification_report, accuracy_score def randomForest(X_train, X_test, y_train, y_test, n_estimators=100, random_state=42): print("\n========== RANDOM FOREST MODEL BUILDING ==========") # 1. Initialize Random Forest Classifier model = RandomForestClassifier( n_estimators=n_estimators, random_state=random_state) print(f"[INFO] RandomForestClassifier initialized with n_estimators={n_estimators}") # 2. Train the model model.fit(X_train, y_train) print("[INFO] Model training completed.") # 3. Predict on test data y_pred = model.predict(X_test) print("[INFO] Prediction completed on test data.") # 4. Evaluate the model accuracy = accuracy_score(y_test, y_pred) cm = confusion_matrix(y_test, y_pred) cr = classification_report(y_test, y_pred) # 5. Display results print(f"\n[RESULT] Accuracy : {accuracy:.4f}") print("\nConfusion Matrix:"); print(cm) print("\nClassification Report:\n"); print(cr) # 6. Return the model and predictions return model, y_pred # model_rf, y_pred_rf = randomForest(X_train, X_test, y_train, y_test)
Story 4.3

K-Nearest Neighbors — KNN()

KNN is a lazy learner that stores the training set and classifies new points based on the majority class among the k nearest neighbours using Euclidean distance. n_neighbors=5 instructs the algorithm to vote among the 5 closest training samples.

  • 1Function KNN() defined. Accepts X_train, X_test, y_train, y_test.
  • 2KNeighborsClassifier(n_neighbors=5) initialized — k=5 means 5 nearest neighbours vote.
  • 3.fit() stores training data — maps input features to target labels.
  • 4.predict() evaluates unseen test data — generates predicted outcomes.
  • 5Accuracy score, confusion matrix, classification report printed. Returns model & predictions.
from sklearn.neighbors import KNeighborsClassifier from sklearn.metrics import confusion_matrix, classification_report, accuracy_score def KNN(X_train, X_test, y_train, y_test): print("\n========== KNN MODEL BUILDING ==========") # Initialize KNN classifier model = KNeighborsClassifier(n_neighbors=5) # Train the model model.fit(X_train, y_train) print("[INFO] KNN model training completed.") # Predict on test data y_pred = model.predict(X_test) print("[INFO] Prediction completed on test data.") # Evaluate model performance accuracy = accuracy_score(y_test, y_pred) cm = confusion_matrix(y_test, y_pred) cr = classification_report(y_test, y_pred) # Display results print("\n[RESULT] Accuracy:", accuracy) print("\nConfusion Matrix::"); print(cm) print("\nClassification Report:"); print(cr) # Return model and predictions return model, y_pred # knn_model, knn_pred = KNN(X_train, X_test, y_train, y_test)
Story 4.4

XGBoost — Gradient Boosting

The function imports a standard Gradient Boosting classifier and performance metrics from scikit-learn / xgboost. The model is initialized and trained on provided training data using .fit(). Predictions on unseen test data are generated using .predict(). Accuracy score, confusion matrix, and classification report are calculated, printed, and the trained model with predictions is returned.

  • 1XGBClassifier imported and initialized with hyperparameters.
  • 2.fit() trains the boosting ensemble — each tree corrects previous errors.
  • 3.predict() generates outcomes on unseen test data.
  • 4Accuracy, confusion matrix, classification report calculated and printed.
  • 5Trained model and predictions returned for deployment.
from xgboost import XGBClassifier from sklearn.metrics import confusion_matrix, classification_report, accuracy_score def xgboost_model(X_train, X_test, y_train, y_test): print("\n========== XGBOOST MODEL BUILDING ==========") # Initialize XGBoost Classifier model = XGBClassifier( n_estimators=200, max_depth=5, learning_rate=0.1, eval_metric='logloss', random_state=42 ) # Train the model model.fit(X_train, y_train) # Predict on test data y_pred = model.predict(X_test) # Evaluate accuracy = accuracy_score(y_test, y_pred) cm = confusion_matrix(y_test, y_pred) cr = classification_report(y_test, y_pred) print(f"\n[RESULT] Accuracy: {accuracy:.4f}") print("\nConfusion Matrix:\n"); print(cm) print("\nClassification Report:\n"); print(cr) return model, y_pred

Story 4.6   Best Model Selection

Why XGBoost Was Selected

Decision Tree, Random Forest, and XGBoost all achieved 96.55% accuracy. Despite the tied score, XGBoost was selected as the final deployment model.

Generalization

Provides strong generalization and performs exceptionally well on structured tabular data — the type of data used in this flood prediction project.

Algorithm Mechanics

As a boosting algorithm, it combines multiple weak learners step by step, reducing errors by learning from previous mistakes in each round.

Stability

More stable and less prone to overfitting compared to a single Decision Tree, while handling complex patterns better than traditional models.

Deployment Readiness

The trained XGBoost model is saved using Joblib. It is loaded by the Flask application and executed to generate real-time predictions from user inputs.

Saving the Final Model

Saving the Best Model with Joblib

After evaluation, XGBoost is saved using the Joblib library. The trained model is stored as floods.save and the fitted StandardScaler as transform.save. Saving both files ensures the same data transformation and prediction logic used during training is applied at deployment. During prediction, the application first loads the scaler to standardize user input, then uses the saved XGBoost model to generate flood predictions.

import joblib # Save the trained XGBoost model joblib.dump(xgb, 'floods.save') # Save the fitted StandardScaler (critical for consistent input transformation) joblib.dump(scaler, 'transform.save') # ─── During Flask deployment: load both files ─── model = joblib.load('floods.save') scaler = joblib.load('transform.save') # Standardize user input using saved scaler, then predict user_input_scaled = scaler.transform(user_input) prediction = model.predict(user_input_scaled)
